# DC-A588D5-V01 规格书深挖

> 依据用户确认：这份 PDF 就是当前板子的正式文档。

## 基本信息

- 型号：`DC-A588D5-V01`
- 版本：`V1.0`
- 板载版本丝印参考：`V01 20251015`
- 厂商：定昌电子（广州市定昌电子有限公司）
- 典型场景：ARM PC、云服务器、智慧大屏、边缘计算、智能 NVR、AR/VR、智能汽车等行业定制市场

## 核心配置

- SoC：Rockchip `RK3588`
- CPU：8 核 64 位（4x Cortex-A76 + 4x Cortex-A55）
- GPU：Mali-G610 MP4
- NPU：6 TOPS
- 内存：LPDDR5，4G / 8G / 16G 可选
- 板载存储：eMMC 16G / 32G / 64G / 128G 可选
- 扩展存储：TF 卡，最高 128GB

## 接口清单（按功能分组）

### 电源 / 按键

- 1x DC 12V 输入（DC 5.5*2.5mm）
- 1x 12V 输入电源插座（2.0mm-4P，备用输入方式）
- 1x 电源键（Power）
- 1x 升级&复位键（短按复位，长按进入升级模式）
- 1x BOOT 键（进入 Loader / Maskrom 的关键按钮）

### 调试 / 串口

- 1x 调试口（`ttys2`），4Pin 白色插座，默认系统调试串口
- 1x TTL（`ttys3`），4Pin 插座，可选配为 RS485
- 2x TTL（`ttys0/ttys7`），可选配为 RS232

### USB / OTG

- 1x USB 3.0 Host（标准 Type-A，下层）
- 1x USB 2.0 OTG（标准 Type-A，上层）
- 4x 2.0mm-4P USB Host 扩展插座（`USB3/USB4/USB5/USB6`）

### 网络

- 2x 千兆以太网口（RJ45），分别为 `网口0` / `网口1`
- 板载 Wi-Fi 6 / BT 5.0
- 2x Wi-Fi 天线座
- 1x Mini-PCIe 4G 模块底座
- 1x SIM 卡座（背面）

### 显示

- 2x HDMI OUT（`HDMI OUT 0` / `HDMI OUT 1`）
- 1x HDMI IN
- 1x LVDS（30Pin）
- 2x EDP（`EDP0` / `EDP1`）
- 1x MIPI DSI（40Pin）

### 音频

- 1x 3.5mm 耳机座
- 1x SPK 功放接口
- 1x 麦克风接口

### 存储扩展

- 1x SATA3.0 数据接口
- 1x SATA 供电接口（提供 +12V / +5V）
- 1x TF 卡槽（正面）
- 1x SIM 卡槽（背面）

### 摄像头

- 2x MIPI CSI（`CSI0` / `CSI1`）

### GPIO / 触控 / 背光 / 其他

- 1x GPIO
- 1x TP（I2C 触摸接口）
- 1x EDP0 背光电源
- 1x LVDS 背光电源
- 1x LVDS/EDP0 屏电源（跳帽可选 3.3V/5V/12V）
- 2x 风扇接口（12V / 5V）
- 1x CAN 接口
- 1x IR 接口

## 调试相关核心信息

### 默认调试串口

- 串口名：`ttys2`
- 说明：文档明确写明“此接口默认为系统调试口”
- 物理形态：4Pin 白色插座
- 位置：SATA3.0 接口下方

### DEBUG / 调试口引脚定义

1. `VCC`（+3.3V 输出）
2. `RX2`
3. `TX2`
4. `GND`

### 建议接线方式

- TTL 模块 `RX` → 板子 `TX2`
- TTL 模块 `TX` → 板子 `RX2`
- TTL 模块 `GND` → 板子 `GND`
- **不要接 VCC**

### 波特率判断

- 文档未明确写波特率
- 按 Rockchip 常见平台经验，优先尝试：
  - `1500000`
  - 不行再试 `115200`
  - 再试 `921600`

## 刷机 / 升级相关核心信息

### USB OTG

- 明确存在：`USB 2.0 OTG`
- 位置：双层 USB Type-A 的**上层**
- 用途：
  - 软件调试
  - 固件升级
  - PC 连接升级

### 升级方式

文档明确支持：

- `PC` 升级
- `U盘` 升级
- `TF卡` 升级

### BOOT / 升级按键语义

- `BOOT`：用于进入 `Maskrom / Loader` 之类的刷机模式
- `升级&复位键`：
  - 短按：复位
  - 长按：进入升级模式

### 推测的实操方式

#### PC 刷机（Maskrom 路线）

1. 断电
2. USB 线连接 **USB OTG 上层口** 到电脑
3. 按住 `BOOT`
4. 再上电
5. 电脑端 Rockchip 工具应识别到 Maskrom / Loader 设备

#### PC 刷机（Loader / 升级模式路线）

1. 开机状态连接 OTG 到电脑
2. 长按“升级&复位键”
3. 尝试进入升级模式

> 注意：现阶段先只验证“能否识别设备”，不要贸然刷写。

## 双网口信息

- 文档明确有 **2 个独立千兆网口**
- 编号明确为：
  - `网口0`
  - `网口1`

这与当前实测现象高度一致：

- 一个口拿到 `192.168.31.176`
- 另一个口拿到 `192.168.31.196`

说明两个物理口大概率都真实可用，且可能对应不同接口 / MAC。

## 与当前已观察现象的对应关系

### 1. 双网口拿到不同 IP

**文档完全支持这一现象。**

- 板子确实有 2 个独立 RJ45 网口
- 后续应在串口里确认每个口对应的接口名、MAC、驱动加载情况

### 2. USB-A 无 ADB

**文档给出了很强的解释。**

- 板子有 Host 口，也有 OTG 口
- 之前大概率插到的是 Host 口，或插到了未开启 gadget/ADB 的口
- 真正应该重点尝试的是：双层 Type-A 的**上层 OTG**

### 3. DEBUG 4Pin

**文档完全确认。**

- 该 4Pin 就是默认系统调试口 `ttys2`
- 可以直接用于抓启动日志

### 4. BOOT 键

**文档完全确认。**

- 这不是普通按键，而是非常关键的刷机模式入口

### 5. TF + SIM

**文档完全确认。**

- 正面是 TF
- 背面是 SIM
- 之前把它误认为“像 USB 的插卡器”，现在已校正

## 当前最实用的行动建议（按优先级）

### 现在就能做

1. 再次确认双层 Type-A 接口的上下层区别
   - 上层：OTG
   - 下层：USB 3.0 Host
2. 在不刷机的前提下，尝试仅做“OTG 枚举验证”
   - 用数据线连接 OTG 上层口到电脑
   - 看电脑是否识别新设备
3. 分别记录 `网口0 / 网口1` 对应拿到的 DHCP 地址
4. 给仓库补板子实拍图，并标出：
   - DEBUG
   - BOOT
   - 升级&复位键
   - OTG 上层口
   - TF / SIM

### 等 USB-TTL 到货后

5. 接通 `ttys2` 调试口，抓完整上电日志
6. 确认：
   - 是否存在 U-Boot
   - 是否有 autoboot 倒计时
   - 启动介质（eMMC / TF / SATA）
   - 网口枚举信息
   - 是否可进入 shell / recovery
7. 再验证 BOOT + OTG 能否被 Rockchip PC 工具识别

## 现阶段结论

这块板子的关键路径已经很明确：

1. **DEBUG 4Pin / ttys2**：最稳的调试入口
2. **USB OTG 上层口**：最可能的 PC 调试 / 升级入口
3. **BOOT 键**：进入刷机模式的关键按键
4. **TF 卡**：可作为升级或恢复路径之一

因此，后续排查路线应围绕：

- 串口日志
- OTG 枚举
- BOOT 模式识别
- 双网口接口映射

而不是继续把普通 Host USB 口当 ADB 调试口反复尝试。
